## Назначение документа

Краткая техническая документация по странице `blog/templates/index.html` (главная страница Helpdesk). Документ описывает архитектуру шаблона, используемые технологии и правила контент‑генерации. Цель — подготовить основу для составления ТЗ нейросети на генерацию нового контента для этой страницы.

## Технологии и стек

- **Flask**: серверный фреймворк Python; страница отдается из блюпринта `main` (маршруты в `blog/main/routes.py`).
- **Jinja2**: шаблонизатор Flask. Страница расширяет базовый шаблон: `{% extends 'layout.html' %}` и определяет блок `{% block main_page %}...{% endblock %}`.
- **HTML5/CSS3**: современная разметка и стили. Классы секций: `hero-section-modern`, `services-section`, `services-grid`, `service-card`, и т.д.
- **Font Awesome**: иконки посредством классов `fas fa-...`.
- **Flask url_for**: генерация ссылок на маршруты и статические ресурсы, например `url_for('main.write_tech_support')`, `url_for('static', filename='img/tez-tour-mini-logo1.png')`.
- **Статические ассеты**: изображения и стили находятся в `blog/static/...`.

## Архитектура и структура шаблона

- **Базовый макет**: наследование от `layout.html` обеспечивает подключение общих стилей, шапки, мета‑данных и скриптов.
- **Hero‑секция**: крупный логотип, заголовок «TEZ TOUR HELPDESK», подзаголовок и декоративные фоновые элементы (`floating-element-*`).
- **Секция «Общие вопросы и поддержка»**: сетка карточек с навигацией к ключевым сценариям поддержки:
  - Обращение в техподдержку (`main.write_tech_support`)
  - Подозрительное письмо? (`main.questionable_email`)
  - Удалённое подключение специалиста (`main.remote_connection`)
  - Проблемы с загрузкой сайтов (`main.no_access_site`)
  - Безопасное использование интернета (`main.safe_internet`)
- **Секция «Настройки ПО и сервисов»**: карточки для настройки VPN, почты, переадресации, адресной книги, Vacuum‑IM, автоответчика:
  - `main.ciscoanyconnect`, `main.setup_mail_account`, `main.setup_mail_forwarding`, `main.adress_book`, `main.vacuum_setup`, `main.auto_resp`
- **Секция «Корпоративные ресурсы»**: карточки для внутренних систем:
  - `main.tez_cloud`, `main.email_setup`, `main.vdi`
- **Футер**: современный футер с текущим годом и контактной почтой.

## Принципы верстки и UX

- **Карточки‑ссылки**: каждая карточка — кликабельная область с иконкой, заголовком, кратким описанием и стрелкой навигации.
- **Семантика**: заголовки секций `h2`, заголовки карточек `h3`.
- **Адаптивность**: сетка карточек реализована классами `services-grid` и `service-card` (реальные медиазапросы находятся в подключенных стилях темы).
- **Иконографика**: единообразные иконки Font Awesome для быстрого распознавания категорий.

## Зависимости от бекенда

- Все ссылки формируются через `url_for`, маршруты должны существовать в блюпринте `main`.
- Изображения и стили загружаются из `blog/static/` через `url_for('static', ...)`.

## Нефункциональные требования

- **Производительность**: минимизировать количество тяжелых изображений; использовать кэширование статики (на уровне сервера/NGINX).
- **Доступность**: у изображений должен быть `alt`; контраст текста соответствовать WCAG AA; интерактивные элементы — в виде ссылок/кнопок.
- **Локализация**: текущий текст — на русском; при генерации контента учитывать корпоративный стиль.

## Контентная модель страницы

Каждая карточка состоит из:

- **title**: короткий, 2–5 слов.
- **description**: 1 короткое предложение (до 90 символов), конкретизирует действие/пользу.
- **icon**: имя класса Font Awesome (`fas fa-...`).
- **route_name**: имя маршрута Flask (`main.<endpoint>`).

Пример структуры данных карточки для генерации (псевдо‑JSON):

```json
{
  "title": "Обращение в техподдержку",
  "description": "Создать новое обращение или проверить статус существующего",
  "icon": "fas fa-headset",
  "route_name": "main.write_tech_support",
  "section": "support"
}
```

## Требования к новому контенту (для ТЗ нейросети)

- **Сохранить структуру**: три секции (Поддержка, Настройки ПО и сервисов, Корпоративные ресурсы) + футер. Допускается добавление новых секций, но с теми же паттернами (заголовок, подзаголовок, сетка карточек).
- **Единый тон**: деловой, короткие формулировки, без маркетингового перегруза.
- **Иконки**: только из Font Awesome, набор `fas` (solid). Подбирать устойчивые ассоциации (например, безопасность — `fa-shield-alt`).
- **Длина**: заголовки карточек до 35 символов; описания до 90 символов.
- **Маршруты**: использовать существующие `main.*` или предлагать новые, но перечислить их явно в итоговом брифе для последующей реализации в Flask.
- **SEO/семантика**: не использовать дубликаты заголовков в пределах секции; указывать говорящие глаголы в названиях.

## Шаблон брифа для генерации контента (скопируй и заполни)

```
Задача: Сгенерировать обновленный контент для главной страницы Helpdesk (`index.html`).

Ограничения стиля:
- Секции: «Общие вопросы и поддержка», «Настройки ПО и сервисов», «Корпоративные ресурсы» (допускается 1 новая секция с аналогичной структурой).
- Заголовки до 35 символов, описания до 90 символов.
- Иконки — Font Awesome (solid), классы `fas fa-...`.
- Тон: деловой, ясный.

Вывести результат в формате JSON массива карточек:
[
  {
    "section": "support|software|corporate|<новая>",
    "title": "...",
    "description": "...",
    "icon": "fas fa-...",
    "route_name": "main.<endpoint>"
  }
]

Требования:
- Не дублировать существующие карточки; предложить 2–4 новых релевантных карточки на секцию.
- Если предлагаются новые маршруты, перечислить их в отдельном массиве `routes_to_add` с кратким описанием назначения.
```

## Пункты проверки после генерации контента

- **Маршруты существуют** или добавлены в `blog/main/routes.py`.
- **Иконки валидны** (проверить в Font Awesome).
- **Длины строк** соответствуют ограничениям.
- **Отсутствуют дубль‑карточки** по названию/назначению.

## Связанные файлы

- Шаблон: `blog/templates/index.html`
- Базовый макет: `blog/templates/layout.html`
- Статика: `blog/static/` (изображения, стили, иконки)
- Маршруты: `blog/main/routes.py`
